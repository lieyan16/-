# 资产管理系统 - 局域网同步协议 v1.0 (草案)

## 概述
本文档定义了手机端(App)与电脑端(Web/Server)在局域网内进行数据同步的通信规则。

## 核心流程
1.  **连接发现**：手机App通过用户手动配置或自动发现，获取电脑主机的IP地址和端口。
2.  **握手认证**：建立连接后，可进行简单的设备标识交换（可选）。
3.  **数据同步**：采用“基于操作日志的增量同步”策略，而非全量覆盖文件。

## 同步API端点示例（设想）
- `POST /api/sync/push`：手机端将本地累积的 `sync_log` 发送给电脑端。
- `POST /api/sync/pull`：手机端从电脑端拉取自上次同步后的新变更。
- `GET /api/info`：获取电脑端的基础信息（如最后同步时间）。

## 冲突解决策略
**策略**：无冲突合并，基于时间戳的“最后写入获胜”(Last Write Win)。
**具体规则**：对于同一资产(`id`)的多次修改，在任何一端合并数据时，无条件保留 `updated_at` 值更大的那条记录。
